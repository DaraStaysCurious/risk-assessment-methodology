## Executive Summary

The documents in this repository make an argument: that OHS risk
assessment methodology, when translated into GRC practice, produces
more accurate risk assessments, more effective controls, and more
durable security programs than framework compliance alone.

The portfolio projects below are where that argument is demonstrated
in practice — not just described.

## Methodology Principles by Project

**Where each principle from this repository appears in practice:**

| Principle | Where it's demonstrated |
|-----------|------------------------|
| OHS hazard ID → GRC threat identification | `grc-occusafe-iso27001` — threat identification methodology explicitly draws on OHS hazard identification practice |
| Fine-Kinney → NIST 800-30 risk scoring | All audit projects — three-variable scoring model applied throughout |
| Five Whys root cause analysis | All audit projects — every corrective action traces to root cause before remediation is designed |
| Human factors layer | `grc-trustpath-usable-security` — dedicated human risk assessment document |
| Friction-kill analysis | `grc-payguard-pcidss` — payment security controls assessed for friction cost before finalization |
| Co-design methodology | `grc-carestream-soc2` — controls developed in dialogue with clinical workflow constraints |
| Design Thinking + Secure Design | `secure-design-framework` — full framework built at the intersection |
| Multi-framework methodology convergence | `grc-meridian-capstone` — all threads unified |
